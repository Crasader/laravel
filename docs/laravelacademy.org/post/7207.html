如果在所有 Laravel 项目中只安装一个扩展包，你会选择哪一个？

发布于 2017-05-24 2017-05-24 23:38:43 学院君 5314 Views

Mike Erickson (@codedungeon) 在 Twitter 上提出了这个问题，吸引了很多回复。我个人感觉由 Barry vd. Heuvel 开发的 Debugbar 将会获取最高票，结果果不其然，它获取了最多推荐。

下面是人们推荐的扩展包列表：

Debugbar

Laravel Debugbar 是一个让你在开发期间可以便捷查看应用所有信息的扩展包，以方便调试。

IDE Helper

该扩展包生成一个 IDE 可以理解的文件，以便提供精准的代码补足、自动完成功能。文件生成基于项目中的文件，所以可以实时更，保持与最新代码同步。

Guzzle

Guzzle 是一个 PHP HTTP 客户端，致力于让发送 HTTP 请求以及与 Web 服务进行交互变得简单。

Spatie Backup

该 Laravel 扩展包用于为应用创建备份，备份文件是一个zip格式文件，包含指定目录下的所有文件以及数据库数据文件，该备份可以存放在 Laravel 中配置的任意文件系统中。

Laravel Collective

Laravel Collective 是一个社区组织，专门维护已经从 Laravel 框架核心移除的组件。

Socialite

Laravel Socialite 提供了一个优雅的流式接口，以便通过 Facebook、Twitter、Google、LinkedIn、GitHub 以及 Bitbucket 等社交媒体进行 OAuth 认证，该扩展包几乎囊括了所有主流的社交媒体认证代码。

Laravel Fractal

该扩展包为在 Laravel 应用中使用 Fractal 提供了一个便捷的封装。

这是国外比较流行的 Laravel 扩展包。你们在自己的 Laravel 应用中都使用了哪些扩展包呢？欢迎留言。
