极限验证，一句话调用geetest，减少开发时间

发布于 2016-06-14 2016-06-14 22:35:06 genar 4570 Views

1、说明

官方给出的sdk调用方法过于浪费开发时间，我就写了一个调用非常简单的一个geetest包尽可能的减少开发时间，如果你觉得官方给出的sdk已经够好了，大可不必使用。

2、安装

  1. composer require laraveler/geetest
  2. 在config/app.php的 providers 数组里加入geetest\provider\geetestServiceProvider::class
  3. 执行php artisan vendor:publish

3、调用

  1. use geetest\Facades\geetest
  2. return geetest::init($request,function(){Auth::attempt([...])},function(){...});

初始化时需要注入$request对象，第二个参数为验证成功后需要做的动作，第三个参数为验证失败需要的动作

注意:调用geetest的方法必须在routes必须注册为any

config在laravel的config目录里（geetest.php）

"api_url"=>"http://api.geetest.com/register.php?gt={ID}&user_id={user_id}"

这里的{ID}经过解析会自动获取config里的ID选项，user_id也是如此。
